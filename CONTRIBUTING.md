## How Can I Contribute?

### Reporting Bugs

If you find a bug, please [open an issue](https://github.com/PUSHPAK-JAISWAL/Document_Summarizer/issues) and include:
- A clear and descriptive title.
- Steps to reproduce the issue.
- Expected and actual behavior.
- Screenshots or logs, if relevant.

### Suggesting Enhancements

We welcome suggestions that improve the project! To propose an enhancement:
- Search existing issues to avoid duplicates.
- [Open a new issue](https://github.com/PUSHPAK-JAISWAL/Document_Summarizer/issues) describing your idea clearly.
- Provide use cases and possible implementation details.

### Submitting Pull Requests

1. **Fork** the repository and clone your fork locally.
2. **Create a branch** for your changes (`feature/my-feature` or `fix/bug-description`).
3. **Write clear, concise code** following the [Style Guide](#style-guide).
4. **Test** your changes.
5. **Commit** your changes with a descriptive message.
6. **Push** to your fork and open a **pull request** on GitHub.

#### Pull Request Checklist

- Provide a descriptive title and summary.
- Reference related issues (if any).
- Ensure your code passes all tests and lints.
- Document any new features or changes.

## Development Setup

1. Ensure you have [Node.js](https://nodejs.org/) and [npm](https://www.npmjs.com/) installed.
2. Clone the repository:
   ```sh
   git clone https://github.com/PUSHPAK-JAISWAL/Document_Summarizer.git
   ```
3. Install dependencies:
   ```sh
   npm install
   ```
4. Run the development server:
   ```sh
   npm start
   ```

## Style Guide

- Use **TypeScript** for all source files.
- Follow [Prettier](https://prettier.io/) formatting.
- Use [ESLint](https://eslint.org/) to maintain code quality.
- Write clear, descriptive comments where necessary.
- Use meaningful variable and function names.

## Commit Message Guidelines

- Use the present tense ("Add feature" not "Added feature").
- Limit the subject line to 50 characters.
- Reference issues when relevant (e.g., `Fixes #11`).
